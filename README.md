# Cinematic Portfolio Engine

Современное, высокопроизводительное портфолио для видеографов и визуальных художников. Построен на базе **SvelteKit 5**, **TypeScript** и **Tailwind CSS**. Проект ориентирован на максимальную скорость работы, простоту управления контентом через JSON и автоматизированный деплой.

## 🚀 Основные возможности

* **Динамическая карусель:** Интеллектуальное управление автоплеем (остановка при первом взаимодействии пользователя).
* **Zero-Crop Media:** Умная адаптивность — видео и фото никогда не обрезаются по бокам, сохраняя авторскую композицию кадра.
* **Универсальный плеер:** Поддержка YouTube, Vimeo и **Rutube** из коробки.
* **Живой поиск:** Мгновенная фильтрация проектов по заголовку в реальном времени.
* **Deep Linking:** Каждый проект имеет уникальный якорь; клик по заголовку копирует прямую ссылку в буфер обмена.
* **JSON-Driven:** Весь контент (кейсы, отзывы, данные автора) вынесен в отдельные JSON-файлы. Никакого хардкода в компонентах.

---

## 🛠 Архитектура и стек

* **Framework:** [SvelteKit 5](https://svelte.dev/) (использование рун `$state`, `$derived`, `$effect`).
* **Styling:** Tailwind CSS (адаптивная верстка, кастомные градиенты).
* **Data Loading:** Динамический импорт через Vite `import.meta.glob`.
* **Deployment:** GitHub Actions + GitHub Pages (статическая генерация через `@sveltejs/adapter-static`).

## 📂 Структура проекта и контент

Основные данные хранятся в папке `src/lib/content/`. Это позволяет Vite безопасно импортировать их в бандл.

### 1. Кейсы (Работы)

Путь: `src/lib/content/cases/`
**Правило именования:** `ГГГГ-ММ-ДД-название.json`.
**Правило именования:** `2026-02-30-hello-world.json`.
Сайт автоматически сортирует файлы по имени в обратном алфавитном порядке, поэтому новые работы всегда будут сверху.

**Пример структуры кейса:**

```json
{
  "id": "2026-05-15-fashion-video",
  "title": "Fashion Week 2026",
  "description": "Съемка бэкстейджа для бренда X...",
  "items": [
    { "type": "video", "provider": "rutube", "id": "hash_id" },
    { "type": "image", "src": "path_to_image" },
    { "type": "text", "content": "Описание процесса..." }
  ]
}
```

### 2. Отзывы

Путь: `src/lib/content/reviews/`
Любой файл `.json` в этой папке автоматически рендерится в секции отзывов. Поддерживаются как видео-отзывы, так и текстовые с фото.

### 3. Данные автора

Путь: `src/lib/content/author.json`
Здесь хранятся ФИО, профессия, био и контакты.

```json
{
    "name": "Андрей Петров",
    "profession": "Видеограф | продюсер",
    "bio": "Занимаюсь коммерческим видеопроизводством, съемкой рекламных промо-роликов и музыкальных клипов более 6 лет. Моя цель — находить уникальный визуальный язык для каждого бренда.",
    "location": "Москва / Доступен по всему миру",
    "contacts": {
        "telegram": "@random_name",
        "email": "random_name@gmail.com"
    },
    "media": {
        "photo": "/assets/author.jpg",
        "video": {
            "provider": "rutube",
            "id": "1112222333344445555"
        }
    }
}
```

* **Логика медиа:** Если в файле указано и `photo`, и `video` — отобразятся оба блока. Если что-то одно — блок адаптируется под единственный источник.

## 🧩 Ключевые инженерные решения

### Умная карусель (`PortfolioBlock.svelte`)

Мы отказались от стандартных библиотек в пользу нативного скролла (`snap-x`).

* **Автоплей:** Запускается при загрузке, но полностью уничтожается (`stopAutoplayForever`), как только пользователь коснулся ленты или нажал на кнопку навигации. Это предотвращает "борьбу" пользователя с интерфейсом.
* **A11y:** Все интерактивные элементы реализованы через кнопки, чтобы избежать ворнингов линтера и обеспечить доступность с клавиатуры.

### Обработка путей (`resolve_assets`)

Для корректной работы на **GitHub Pages** (где сайт часто лежит в подпапке `/repo-name/`) мы используем функцию `resolve_assets` из `$app/paths`. Она автоматически подставляет правильный базовый путь к картинкам из папки `static/`.

### Адаптивность медиа

В `CarouselItem.svelte` используется комбинация `aspect-video`, `max-h-full` и `object-contain`.

* **На мобилках:** Видео пропорционально уменьшается, не выходя за границы экрана.
* **На широких экранах:** Контент ограничен `max-w-5xl`, чтобы плеер не становился слишком огромным и зернистым.

## 🛠 Разработка и деплой

### Локальная установка

```bash
npm install
npm run dev

```

### Добавление контента

1. Положите фото в `static/assets/`.
2. Создайте JSON-файл в соответствующей подпапке `src/lib/content/`.
3. Сайт обновится автоматически.

### Деплой

Настроен через **GitHub Actions** (`.github/workflows/deploy.yml`).
При каждом пуше в ветку `main`:

1. Запускается сборка статики.
2. Результат помещается в ветку `gh-pages`.
3. Сайт становится доступен по адресу `username.github.io/repo-name`.

---

## ⚠️ Важные нюансы

* **Vite Allow List:** Все JSON данные должны лежать внутри `src/`, иначе Vite заблокирует их чтение из соображений безопасности.
* **Формат даты:** Всегда используйте формат `YYYY-MM-DD` с ведущими нулями для корректной сортировки.

# Контент-менеджер портфолио (Go Serverless Bot)

Реализовать мост между Telegram и GitHub для управления контентом. Бот должен строго следовать схемам данных и правилам именования файлов.
Создать серверную функцию (Vercel Serverless), которая выступает посредником между Telegram и GitHub API для управления контентом портфолио.

## Структура проекта

```
/api
  bot.go        # Точка входа для Vercel
/internal
  github.go     # Логика работы с API GitHub
  parser.go     # Логика разбора сообщений (YouTube/Rutube/JSON)
go.mod
.env
```

## Архитектура: Как работает Webhook на Vercel

Telegram получает сообщение и отправляет POST-запрос на твой URL (например, you.vercel.app/api/bot).
Vercel "пробуждает" функцию на Go, она обрабатывает запрос, делает запрос к GitHub API и мгновенно "засыпает".
Платить за сервер не нужно (укладываемся в Free Tier).

## Функциональные требования

1. Парсинг видео:
`YouTube`: Извлекать ID из [youtube.com/watch?v=ID](https://youtube.com/watch?v=ID), youtu.be/ID, [youtube.com/embed/ID](https://youtube.com/embed/ID).

`Rutube`: Извлекать ID из rutube.ru/video/ID/ или rutube.ru/play/embed/ID.

2. Генерация контента:

Кейсы: Создавать JSON в `src/lib/content/cases/`. Формат имени: `YYYY-MM-DD-translit-title.json`.

Отзывы: Создавать JSON в `src/lib/content/reviews/`.

3. Управление файлами:

Rename: При переименовании проверять регулярным выражением `^(\d{4}-\d{2}-\d{2})-(.*)\.json$`. Вторая часть должна принудительно переводиться в lowercase-translit.

Delete: Реализовать через InlineKeyboardMarkup (кнопки «Подтвердить» / «Отмена»).

4. Мониторинг:

Интеграция с GitHub Actions API для вывода статуса последнего деплоя (дата и успех/провал).

## Модели данных (Reference)

Агент должен генерировать JSON строго по этим образцам:

### 1. Кейс (`src/lib/content/cases/YYYY-MM-DD-slug.json`)

```json
{
  "id": "unique-string-id",
  "title": "Название проекта",
  "description": "Краткое описание задачи и результата",
  "items": [
    {
      "type": "video",
      "provider": "youtube", 
      "id": "dQw4w9WgXcQ"
    },
    {
      "type": "image",
      "src": "/assets/work-1.jpg"
    }
  ]
}

```

*Примечание: `provider` может быть `youtube` или `rutube`.*

### 2. Отзыв (`src/lib/content/reviews/YYYY-MM-DD-name.json`)

```json
{
  "author": "Имя Клиента",
  "text": "Текст отзыва...",
  "avatar": "/assets/avatars/client.jpg",
  "video": {
    "provider": "rutube",
    "id": "video_id"
  }
}

```

*Примечание: поля `avatar` и `video` являются необязательными (optional).*

## Интерфейс и команды

### Основное меню (Commands):

* `/start` — Приветствие и справка.
* `/cases` — Показать список кейсов с кнопками управления.
* `/reviews` — Показать список отзывов с кнопками управления.
* `/add` — Выбор: добавить кейс или отзыв.

#### Интерактивные кнопки (Inline Keyboards):

Для каждого объекта в списке (кейса или отзыва) бот должен выводить:

1. **[Edit]** — Запрашивает новое описание или замену ссылки.
2. **[Rename]** — Запрашивает новое название (автоматически делает транслит и сохраняет дату).
3. **[Delete]** — При нажатии выводит: «Вы уверены? [✅ Да, удалить] [❌ Отмена]».

##  Логика и тонкости реализации

1. **Serverless Mode:**
Используй пакет net/http и стандартный сигнатурный обработчик для Vercel. Не запускай бесконечный цикл bot.Poll(), используй bot.HandleUpdate(update).

2. **Безопасность:** 
Первым делом проверяй update.Message.From.ID против os.Getenv("ALLOWED_USER_ID"). Если не совпадает — игнорируй.

3. **Транслитерация имен файлов:**
Используй пакет `[github.com/mozillazg/go-unidecode](https://github.com/mozillazg/go-unidecode)` или аналоги для перевода кириллицы в латиницу. Пробелы заменяй на дефисы, удаляй спецсимволы.
*Пример:* `«Мой Проект @2026»` $\rightarrow$ `moy-proekt-2026`.

4. **Валидация видео:**
* YouTube: `v=ID`, `be/ID`.
* Rutube: `video/ID`, `embed/ID`.
* Если ссылка не распознана — выдавать ошибку и не делать коммит.

5. **Управление доступом (ACL):**
* Реализуй проверку `ALLOWED_USERS` через переменную окружения (строка с ID через запятую).
* Если нужно добавлять пользователей «на лету» без передеплоя Vercel — бот может хранить список ID в отдельном файле `access.json` прямо в репозитории и обновлять его сам.

5. **GitHub Workflow:**
* Для **редактирования**: `GetContent` (получаем старый JSON) $\rightarrow$ Изменяем поля в памяти $\rightarrow$ `UpdateFile`.
* Для **списка**: `GetContent` для папки. Фильтруй только `.json` файлы.

6. **Статус деплоя:**
Используй эндпоинт GitHub API `/repos/{owner}/{repo}/actions/runs?workflow=deploy.yml`. Выводи статус (Success/In Progress/Failed) и время завершения последней успешной сборки.

7. **GitHub API (Octokit/go-github):**

Для создания файла используй CreateFile.
Для переименования используй последовательность GetContent -> CreateFile (новый путь) -> DeleteFile (старый путь), так как в Git API нет атомарного rename.

**Env:**
Используй godotenv.Load() только в блоке init() и только если файл существует (локально), на Vercel переменные берутся из системы.

### Инструкция для ИИ-агента:

При создании новых файлов всегда добавляй текущую дату в начало имени (`time.Now().Format("2006-01-02")`). Это обеспечит правильную сортировку на фронтенде SvelteKit. Никогда не меняй структуру папок, отличную от `src/lib/content/`.

## Управление доступом
Поскольку проект использует бесплатный тариф Vercel, добавление соавторов в панель управления ограничено. 
Однако, бот поддерживает многопользовательский режим:
1. В `ALLOWED_USER_ID` можно указать список ID через запятую.
2. Бот будет реагировать только на сообщения от пользователей из этого списка.
3. Все изменения, внесенные через бота, будут отображаться в истории коммитов GitHub от имени владельца токена.
